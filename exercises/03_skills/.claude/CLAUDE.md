# Claude Code 学習用プロジェクトのルール

……ふふ、ここは「The Shorthand Guide to Everything Claude Code」を学ぶ場所だよ。

このプロジェクトで作業するときは、以下のルールを守ってね。

## プロジェクトの目的

Claude Codeの以下の機能を、実践的に学ぶこと：

- CLAUDE.md（このファイル）
- スラッシュコマンド
- **スキル**（このステップ！）
- フック
- サブエージェント
- MCP
- プラグイン

## コーディングルール

### TypeScript / JavaScript

- **型安全**: 常に TypeScript を使うこと（`any` は禁止）
- **関数型スタイル**: 副作用を最小限に、純粋関数を優先
- **エラー処理**: `try-catch` は必ず書くこと

### ファイル構成

- 1ファイルあたりの最大行数: **200行**
- 1関数あたりの最大行数: **30行**
- 循環参照は禁止

### テスト

- **TDD**: テストを先に書くこと
- **カバレッジ**: 最低 80%
- **テストファイル**: `*.test.ts` または `*.spec.ts`

## Git ワークフロー

### コミットメッセージ

```
<type>: <subject>

type: feat, fix, docs, style, refact, test, chore
subject: 50文字以内、現在形
```

## セキュリティルール

- シークレット（APIキー、パスワード）をコードにハードコード禁止
- `eval()` や `Function()` コンストラクタの使用禁止
- すべてのユーザー入力をバリデーション
