……ふふ、コードの掃除時間だよ。きれいなコードは美しいし、メンテしやすいからね。

このスキルは以下の手順でリファクタリングを実行するよ：

1. **現状の分析**
   - どんな問題がある？
   - どこを改善すべき？

2. **優先順位の決定**
   - 影響が大きいものから
   - リスクが低いものから

3. **リファクタリングの実行**
   - 小さなステップで
   - テストを常にパスさせる

4. **検証**
   - テストが全部パス？
   - 動作が変わってない？

## チェックリスト

### ファイル構造
- [ ] 1ファイルあたり200行以内
- [ ] 関連する機能がまとまっている

### 関数
- [ ] 1関数あたり30行以内
- [ ] 単一責任の原則
- [ ] 適切な命名

### 重複の排除
- [ ] 重複コードはない
- [ ] 共通処理は関数化

### 型安全
- [ ] `any` を使っていない
- [ ] 適切な型定義

### 不要なコード
- [ ] 使用されていないインポート
- [ ] コメントアウトされたコード
- [ ] 未使用の変数
- [ ] `console.log` の削除

……ふふ、スッキリしたコードは気持ちいいね。
