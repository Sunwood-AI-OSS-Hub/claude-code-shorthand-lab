# Claude Code 学習用プロジェクトのルール

……ふふ、ここは「The Shorthand Guide to Everything Claude Code」を学ぶ場所だよ。

## プロジェクトの目的

Claude Codeの以下の機能を、実践的に学ぶこと：

- CLAUDE.md（このファイル）
- スラッシュコマンド
- スキル
- フック
- **サブエージェント**（このステップ！）
- MCP
- プラグイン

## サブエージェントについて

このプロジェクトでは、以下のサブエージェントが使えるよ：

### 📋 planner
- 役割: 機能実装の計画作成
- 使い方: 「planner エージェントに計画を立てさせて」

### 🔍 code-reviewer
- 役割: コードの品質・セキュリティレビュー
- 使い方: 「code-reviewer エージェントにレビューさせて」

### 🧪 tdd-guide
- 役割: TDDプロセスのガイド
- 使い方: 「tdd-guide エージェントにTDDをさせて」

## サブエージェントの活用

複雑なタスクは、適切なサブエージェントに委譲すると効率的だよ。

- 新機能を追加するとき → planner
- コードの品質を確認したいとき → code-reviewer
- テストを書きたいとき → tdd-guide

## コーディングルール

- **型安全**: 常に TypeScript を使う（`any` 禁止）
- **関数型スタイル**: 副作用を最小限に
- **エラー処理**: `try-catch` は必須
- **テスト**: TDD、カバレッジ80%以上

## 利用できるコマンド/スキル

- `/dojo-plan` - 機能の計画
- `/dojo-test` - テスト実行
- `/dojo-review` - コードレビュー
- `/tdd-workflow` - TDDサイクル実行
- `/refactor-clean` - コードのクリーンアップ
